## 📌 核心发现

### 🐛 已修复的关键 Bug (3个)

1. **Telemetry 类型不匹配** (`telemetry/client.go:731`)
   - 症状: UPDATE 操作失败，`auto_decision` 列类型错误
   - 影响: 所有 telemetry 更新失败
   - 修复: 移除不必要的 CAST，直接传递 TEXT
   - 状态: ✅ 已修复

2. **TimescaleDB 约束不匹配** (`telemetry/client.go:989`)
   - 症状: UPSERT 失败，`ON CONFLICT` 约束不存在
   - 影响: 无法更新已存在的 request_logs 记录
   - 修复: 改为 `ON CONFLICT (request_id, ts)` 匹配分区表约束
   - 状态: ✅ 已修复

3. **no_candidate 路径缺少 WAL** (`relay/handler.go:976`)
   - 症状: no_candidate 错误的请求不写入 request_wal
   - 影响: 无法追踪所有失败请求
   - 修复: 添加 `requestLogger.CreateInitial` 调用
   - 状态: ✅ 已修复

## ⚠️ 测试限制

### 加密凭据系统依赖
**问题**: Gateway 需要解密密钥才能调用上游 API

```
错误: credential decrypt failed: no decryption key available
影响: 无法完成真实的上游调用
```

**导致无法完整测试**:
- ❌ 路由切换（tier 1 → tier 2 fallback）
- ❌ 指纹管理（identity_hash 绑定到 credential）
- ❌ Slot 抢占（fp_slot_limit 限制）
- ❌ 多会话并发
- ❌ 会话内模型变更

### 自动路由索引的循环依赖
Gateway 从**成功的 request_logs** 生成路由索引，但需要路由索引才能成功调用上游：

```
request_logs (成功) → 生成索引 → 路由请求 → 上游调用 → 生成 request_logs
    ↑                                                              ↓
    └───────────── 凭据无法解密导致循环中断 ────────────────────────┘
```
